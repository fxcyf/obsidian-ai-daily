# Feature: 工作流自动化

**状态**: ⬜ 待开始

## 概述

让 AI 在后台主动工作，减少用户手动操作，从"工具"变为"助手"。

---

## 1. 笔记自动标注

**优先级**: P1
**难度**: 中

### 现状
新导入的 Raw 笔记缺少 tags、摘要等 metadata，需要手动补充。

### 方案
- 检测新增/修改的笔记（通过 Obsidian 的 `vault.on('create')` 事件）
- 自动调用 Claude 分析笔记内容，生成：
  - tags（写入 frontmatter）
  - 一句话摘要（写入 frontmatter 的 `summary` 字段）
  - 推荐关联笔记（以 wiki-link 追加到笔记底部）
- 用户可在 Settings 中开关此功能，配置自动标注的文件夹范围

### 实现要点
- 监听文件创建事件，延迟执行（避免编辑中触发）
- 只处理配置范围内的文件夹（如仅 Raw/）
- 避免重复标注：检查 frontmatter 中是否已有 `auto-tagged: true`
- API 调用成本提示：每次标注消耗约 1k tokens

## 2. 定时日报/周报生成

**优先级**: P2
**难度**: 中

### 现状
需要手动触发 Feed 生成，无法自动化。

### 方案
- 利用 Obsidian 的 `onLayoutReady` 在每次启动时检查
- 如果当天尚未生成日报，自动触发生成流程
- 周报：每周一自动汇总过去一周的笔记活动

### 实现要点
- 通过检查 Feed 文件夹中是否存在当天文件来判断
- 后台静默执行，完成后发送 Obsidian Notice 通知
- 可配置：启动时自动生成 / 仅手动触发
- 与 feature-info-feed 的 Feed 生成逻辑复用

---

## 3. 智能回顾提醒

**优先级**: P3（远期）
**难度**: 中

### 概念
- 基于间隔重复原则，定期提醒用户回顾重要笔记
- 结合笔记的创建时间、访问频率、关联度等信号
- 在 Chat 侧边栏中展示"今日回顾"卡片

### 依赖
- 需要对话历史持久化（追踪哪些笔记被讨论过）
- 需要笔记访问记录（可通过 Obsidian API 获取）
